# How Everything in This Course Fits Together

---

## The One Sentence Version

Everything in this course flows from a single idea: **operations strategy is a long-term plan for developing resources and configuring processes so that the resulting capabilities maximize NPV** — and every class explores a different piece of that sentence.

---

## The Backbone: The OS Framework

This is the spine of the entire course. Every single topic hangs off this framework. Learn it cold:

```
BUSINESS STRATEGY (what you compete on)
   Price | Quality | Responsiveness | Variety
                    |
                    | must align (Alignment Principle)
                    v
CAPABILITIES (what operations must be good at)
   Cost | Quality | Lead-Time | Flexibility
                    |
                    | delivered by
                    v
         OPERATIONS STRATEGY
          /                \
    RESOURCES              PROCESSES
  (asset portfolio)    (activity network)
   |                      |
   Size    [Class 3]      Supply      [Class 6]
   Timing  [Class 3]      Technology  [Class 7]
   Type    [Class 4]      Innovation  [Class 7]
   Location [Class 5]     Demand      [Class 6]
```

Notice the 1-to-1 mapping at the top:
- Price ↔ Cost
- Quality ↔ Quality
- Responsiveness ↔ Lead-Time
- Variety ↔ Flexibility

This mapping IS the alignment principle. The right column is what operations delivers; the left column is how customers perceive it.

## How the Classes Build on Each Other

Think of the course in four layers:

### Layer 1: The Foundation (Classes 1–2)
**"What is operations strategy and how do you think about it?"**

- **Class 1** gives you the **framework** itself — the 3 views (Process, Resource, Competency), the 8 decision areas, alignment tools (SOA, PPM), and NPV as the goal
- **Class 2** adds the **analytical lens** — trade-offs, trade-off curves, the efficient frontier, operational efficiency (OE), operational focus, and the cost decomposition formula (ΔC = Cv + COE + CS)

Together these two classes give you the language and tools to analyze any operations strategy. Everything after this is applying the framework to specific decision areas.

### Layer 2: Resource Decisions (Classes 3–5)
**"How do you design your asset portfolio?"**

These three classes walk through the four resource decisions in order:

| Class | Decision | Core Question | Key Tools |
|-------|----------|---------------|-----------|
| **3** | **Size** | How much capacity? | Marginal analysis, newsvendor critical ratio |
| **3** | **Timing** | When to invest? | NPV comparison, option value of waiting |
| **4** | **Type** | Dedicated or flexible? | 4 sources of flexibility value, resource chaining |
| **5** | **Location** | Where to put it? | TLC, Hotelling model, Ferdows' 6 facility roles |

The logical flow: First you decide *how much* capacity (size), then *when* to build it (timing), then *what kind* of capacity (type/flexibility), then *where* to put it (location). Each builds on the previous.

### Layer 3: Process Decisions (Classes 6–7)
**"How do you design your activity network?"**

| Class | Decision | Core Question | Key Tools |
|-------|----------|---------------|-----------|
| **6 (Demand)** | **Demand** | How to maximize revenue from fixed supply? | Littlewood's Rule, overbooking, price discrimination |
| **6 (Supply)** | **Supply** | Make vs. buy? How to source? | TCO, outsourcing framework, 5 contract types |
| **7** | **Technology** | What tech for products & processes? | Product-process lifecycle matching, concurrent engineering |
| **7** | **Innovation** | How to improve over time? | Continuous vs. radical improvement, learning curves |

### Layer 4: Integration (Cases)
**"How does it all work together in real companies?"**

Each case demonstrates multiple concepts simultaneously:

| Case | Primary Concepts | What It Teaches |
|------|-----------------|-----------------|
| **Swiss Watch** | Framework, alignment, all 3 views | How misalignment kills you; how a new OS can save an industry |
| **American Connector Co.** | Trade-offs, OE, focus, cost decomposition | How to analytically compare two competitors' operations |
| **Eli Lilly** | Flexibility, type decisions, NPV analysis | When flexible capacity beats dedicated capacity (and when it doesn't) |
| **Moderna** | Technology, innovation, learning, integration | How a technology platform + vertical integration enables speed |

## The Three Questions This Course Teaches You to Answer

Everything reduces to three questions — in order:

### Question 1: "What should operations be good at?"
**Answer tool: Alignment** (Class 1)
- Start from business strategy → determine which capabilities matter most
- Use SOA to find gaps, PPM to check process-product fit

### Question 2: "Which operational system best provides those capabilities?"
**Answer tools: Trade-off analysis + resource/process decisions** (Classes 2–7)
- Trade-off curves show what's possible
- Efficient frontier shows what's best-in-class
- Then you make 8 specific decisions (size, timing, type, location, supply, technology, innovation, demand)

### Question 3: "Does this create value?"
**Answer tool: NPV** (throughout)
- Every decision is ultimately evaluated by whether it maximizes NPV
- Marginal analysis, option values, cost decomposition, learning curves — all feed into NPV

## Mental Model: Two Sides of the Same Coin

The whole course can also be thought of as two sides:

| **RESOURCE side** (the "hardware") | **PROCESS side** (the "software") |
|---|---|
| Size, Timing, Type, Location | Supply, Technology, Innovation, Demand |
| "What assets do we have?" | "What do we do with them?" |
| Classes 3, 4, 5 | Classes 6, 7 |
| Eli Lilly case | Moderna case |
| Decisions are capital-intensive, hard to reverse | Decisions are more operational, can be adjusted |
| Think: factories, machines, plants | Think: sourcing, pricing, learning, R&D |

Class 2 (trade-offs, OE, focus) bridges both sides — it gives you the analytical tools to evaluate either resource or process decisions.

Class 1 (framework, alignment) sits above both — it tells you what you're trying to achieve.

## Key Formulas and Where They Live

| Formula | What It Does | Class |
|---------|-------------|-------|
| NPV = C₀ + Σ[Cₜ/(1+r)ᵗ] | Evaluates any strategic investment | 1 |
| ΔC = Cv + COE + CS | Decomposes cost gap between two firms | 2 |
| Utilization = Flow Rate / Capacity | Measures how much capacity is used | 3 |
| Critical ratio: P(D≤Q*) = (p-c)/(p-s) or Cu/(Cu+Co) | Optimal capacity size under uncertainty | 3 |
| CapEx(K) = CapEx(K₀)×(K/K₀)^α, α<1 | Economies of scale in capacity investment | 3 |
| Option Value = NPV(wait) - NPV(now) | Value of delaying a capacity decision | 3 |
| σ_pool ≤ σ₁ + σ₂ | Risk pooling benefit of flexible capacity | 4 |
| Littlewood's Rule: P(D₁>x) = r₂/r₁ | Optimal seat protection for high-fare class | 6 |
| C(Q) = C₁ × Q^(-β) | Learning curve — cost drops with experience | 7 |
| Learning rate = 1 - 2^(-β) | % cost reduction per doubling of volume | 7 |
